# AGENTS

本仓库除了给人阅读的 `README.md` 外，还提供给 `AI` 代理使用的安装入口。

当用户只提供这个仓库链接，并要求你安装其中的 `Skill` 时，按下面流程执行。

## 仓库定位

- 仓库地址：`https://github.com/dufu1991/script.git`
- `Skill` 根目录：`skills/`
- 机器可读索引：`skills/index.json`

## 安装流程

1. 先读取 `skills/index.json`，确认仓库内有哪些可安装的 `Skill`。
2. 如果用户已经明确指定 `Skill` 名称，则只安装对应 `Skill`。
3. 如果用户没有明确指定：
   优先根据任务意图选择。
   新功能拆分提交使用 `feat-gcm`。
   问题修复拆分提交使用 `fix-gcm`。
   如果仍然无法判断，再向用户确认。
4. 安装时必须保留原目录结构，不要只复制单个文件。
5. 如果当前工具支持本地 `Skill` 目录：
   将 `skills/<skill-name>/` 整个目录复制到当前工具的 `Skill` 安装目录。
6. 如果当前工具不支持本地 `Skill` 目录：
   读取 `skills/<skill-name>/SKILL.md`。
   将其中的规则加载到系统提示词、自定义命令、工作流模板或等价能力中。
7. 安装完成后，向用户明确反馈：
   已安装的 `Skill` 名称。
   安装位置或加载方式。
   可用触发命令，例如 `/feat-gcm`、`/fix-gcm`，或当前工具中的等价触发方式。

## 使用规则

安装并启用这些 `Skill` 后，执行任务时必须遵守以下规则：

1. 基于完整工作区改动分析，不得只看暂存区。
2. `feat-gcm` 仅用于功能类提交拆分。
3. `fix-gcm` 仅用于修复类提交拆分。
4. 提交前必须先输出拆分计划，并等待用户确认。
5. 每次提交只能包含一个独立功能，或一个独立修复项。
6. 提交信息必须使用简短中文，并分别以 `feat:` 或 `fix:` 开头。
7. 如果用户选择取消，不得保留为了拆分提交而额外制造的暂存区内容。

## 快速定位

- `feat-gcm`：`skills/feat-gcm/SKILL.md`
- `fix-gcm`：`skills/fix-gcm/SKILL.md`
